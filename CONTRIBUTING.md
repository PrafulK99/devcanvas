# 🤝 Contributing to DevCanvas

Thank you for your interest in contributing to **DevCanvas** 🚀
We welcome contributions from everyone — especially beginners!

---

## 🧠 About the Project

DevCanvas is a developer-first visual canvas to:

* Map code logic
* Debug workflows
* Design system architecture

Built using:

* React
* Tailwind CSS v4
* React Flow
* Zustand

---

## ⚡ Getting Started

### 1. Fork the Repository

Click the **Fork** button on GitHub.

---

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/devcanvas.git
cd devcanvas
```

---

### 3. Install Dependencies

```bash
npm install
```

---

### 4. Run the Project

```bash
npm run dev
```

App will run at:
👉 http://localhost:5173

---

## 🌿 Create a Branch

```bash
git checkout -b feature/your-feature-name
```

---

## 🛠️ Make Your Changes

* Follow existing code style
* Keep components clean and reusable
* Test your changes before committing

---

## ✅ Commit Your Changes

```bash
git add .
git commit -m "feat: add your feature name"
```

---

## 🚀 Push and Create PR

```bash
git push origin feature/your-feature-name
```

Then open a Pull Request on GitHub.

---

## 🏷️ Issues

We have categorized issues:

* 🟢 `good first issue` → beginners
* 🟡 `enhancement` → improvements
* 🔴 `feature` → new features

👉 Start with **good first issue** if you're new!

---

## 🧑‍💻 Contribution Guidelines

* Keep PRs small and focused
* Do not break existing features
* Write clear commit messages
* Link your PR to an issue

---

## 💡 Need Help?

Feel free to:

* Ask questions in issues
* Comment before starting work

---

## 🎯 Goal

We aim to make DevCanvas:

* Simple
* Fast
* Developer-friendly

---

Thanks for contributing ❤️
Let's build something awesome together 🚀
